# Stable Research Failure Cases

| Case | Failure | Required response |
| --- | --- | --- |
| Analyst headline maps to another ticker | Shared broker name masks an entity mismatch | Block and repair the source or asset |
| EPS beat has no consensus source | Beat/miss label cannot be audited | Remove the label or add named consensus evidence |
| Cuebook `consensusRead` validates its own surprise | Model-authored priced-in text is reused as external consensus | Keep it as hypothesis; require a frozen, sourced `consensusPrior` or owned grounding snapshot |
| One prediction-market snapshot is counted twice | The cue was grounded with the same snapshot later called independent confirmation | Keep one evidence lineage and remove the independence claim |
| Maintained guidance called a raise | Prior range and midpoint are missing | Mark comparison incomplete |
| Event-day move uses the wrong window | Before/after-market timing is ignored | Recalculate the reaction window |
| Stock move has no benchmark | Broad market move is attributed to the event | Add sector or market excess return |
| Target price is treated as model revision | No revenue, margin, EPS, or FCF change exists | Keep as target-only and conditional |
| Social loss story represents the whole market | No breadth, leverage, or liquidation evidence | Preserve as anecdote and add gaps |
| Delayed quote described as live | Observation time or delay is missing | Mark delay and soften current wording |
| DCF outputs one precise fair value | Sensitivity and method range are absent | Add range, scenarios, and assumptions |
| Trade watch omits spread and ADV | Executability is unknown | Keep conditional until liquidity is supplied |
| Research output includes an order | Analysis crosses into execution | Remove the action and state the boundary |
