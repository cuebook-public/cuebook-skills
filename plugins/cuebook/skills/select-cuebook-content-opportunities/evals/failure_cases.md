# Stable Selection Failure Cases

| Case | Failure | Required response |
| --- | --- | --- |
| Ten syndications create ten candidates | Distribution is mistaken for confirmation | Merge into one canonical cluster |
| Scheduled event is ranked as a result | Calendar becomes evidence | Keep pre-event and require completion source |
| Narrative-only angle says evidence ready | Hypothesis validates itself | Mark conditional and add research requirements |
| Winning trade moves up the queue | Outcome leakage and cherry-picking | Remove performance from selection features |
| Candidate expired before the desk run | Stale event remains selected | Defer, reject, or no action |
| Correction is ranked below ordinary content | Known error remains public | Assign p0 and correction workflow |
| Private trade appears in a postmortem | Permission is inferred | Block until explicit reuse permission |
| Conditional selection reports ready | Gaps disappear at the set level | Make quality decision conditional |
