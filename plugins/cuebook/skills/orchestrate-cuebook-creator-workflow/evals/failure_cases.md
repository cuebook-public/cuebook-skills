# Stable Orchestration Failure Cases

| Case | Failure | Required response |
| --- | --- | --- |
| Render runs after gate reject | Dependency state is ignored | Block render and downstream release |
| Route abstention disappears | Reduced route object bypasses uncertainty | Carry abstention and block readiness |
| Draft changes after approval | Approval points to stale content | Revoke and request new approval |
| HTTP success has no remote identity | Attempt is called publication | Keep ambiguous or acknowledged |
| Correction changes a source fact | Old derivatives remain current | Invalidate dependency subtree |
| Engagement drives research truth | Packaging metric contaminates evidence | Keep outcome planes separate |
| Workflow says complete with deferred publisher | Handoff is confused with completion | Use ready_for_handoff |
| Saved recipe runs against a newer unpinned catalog | Reproducibility is lost | Re-resolve into a new recipe revision |
| Frontend disables an internal evidence skill | Safety dependency is removed | Restore the required skill and block execution |
| Post and visual use different expression fingerprints | Meaning can drift across media | Bind both branches to one CreatorExpressionPlanV1 revision |
| Source-only observation gains a thesis or settlement | Trade intent is invented | Remove trade-only nodes and preserve non-trade semantics |
| Viewpoint visual bypasses data assembly | Displayed primitives lose source and cutoff checks | Add ViewpointDataBundleV1 as an ancestor |
| Conditional research/data is hidden behind a remote artifact locator | Orchestrator cannot enforce material-premise gates without unsafe payload access | Register the normalized gate summary and block advanced render/candidate descendants without dereferencing the locator |
| Candidate completes with an unresolved material request | Selectable output silently drops a creator premise | Transition candidate and dependent release nodes to blocked and retain the blocked set only for inspection |
