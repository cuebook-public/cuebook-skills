# Creator Workflow Rubric

| Dimension | 5 means |
| --- | --- |
| Ownership | Each capability has one skill and one output contract |
| Recipe resolution | User choices, automatic requirements, catalog 1.26.0, transitive dependencies, and exact skill pins agree |
| Semantic bridge | Semantics precede one shared expression plan; post and data branch in parallel and preserve its meaning fingerprint |
| Trade optionality | Source-only non-trade semantics do not acquire trade logic, thesis, or settlement nodes |
| Dependency safety | Gates, research, semantics, expression, render, preflight, release, and receipt order is explicit and acyclic; non-ready gate summaries block advanced render/candidate descendants |
| Lineage | Every artifact has a hash, producer, input set, and current status; gate-bearing artifacts expose normalized summaries without locator dereference |
| State integrity | Append-only events reconstruct every node and blocker |
| Governance | Human decisions bind exact versions and material edits revoke approvals |
| Activation honesty | Prepared, attempted, acknowledged, and verified publication cannot be confused |
| Recovery | Corrections invalidate dependent artifacts and lead to a traceable replacement path |

A ready handoff requires a current approved ReleaseBundleV1 and no blocking node. `complete` requires all blocking work, publication receipt, and applicable reconciliation.
